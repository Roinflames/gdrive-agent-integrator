# AGENTS.md - Shared Workspace Rules

## Persistent Documentation Rule
- Do not leave operational decisions only in chat.
- Any agreement that affects process, access, or team operations must be written to the repository in a tracked file.
- For requests related to Google Drive / Shared Drive, always create or update documentation under `docs/` (or create `docs/` if missing) with:
  - purpose and scope,
  - owners and admins,
  - membership/access matrix,
  - folder structure,
  - naming conventions,
  - review cadence and change log.

## Execution Standard
- If a user asks to "remember", "persist", or "document" a rule, treat it as mandatory repository documentation work.
- Prefer concrete artifacts over chat summaries (markdown files, checklists, runbooks, decision logs).
